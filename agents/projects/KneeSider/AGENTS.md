# AGENTS_PROJECT.md

このファイルは、そのプロジェクト固有の AI agent 向け運用メモです。
共通ルールは dotfiles の `templates/AGENTS_BASE.md` に集約します。

## Goal
- このプロジェクトは膝側面レントゲン写真を対象に、正側面を撮影するための修正角度を推定するツールを開発することを目的としています。
- 方法としては、膝のCT画像から正側面に対する角度パターンもったレイサム画像を作成し、それを教師データとして機械学習モデルを訓練します。

## Context
- 使用するメインの言語はpython。
- reference/2211019.pdfが先行研究だが、学習データの分け方がおそらく不適切であるため、そこの改善を試みている。（リークがある）
- 現状では画像シリーズの数が足りていない。新しくデータを手に入れたが、現在の画像増強用のソフトに入れるには前処理が必要。（画像のトリミングが必要）
- 直近の発表においてはまずは先行研究における学習セットの扱い方を適切なものに変更し、その評価を行うところまでとする。

## Commands
- このディレクトリでよく使うコマンドを書く。
- 各コマンドが何をするかを書く。
- 変更後の確認に使うコマンドがあれば書く。

例:
- `rtk npm test`: テスト実行。
- `rtk make lint`: lint 実行。
- `rtk pytest path/to/test_file.py`: 特定テストの確認。

## Conventions
- 命名、フォーマット、レビュー方針、設計ルールを書く。
- 守るべき既存パターンと、避けるべきパターンを書く。
- フレームワーク固有の流儀があるならここに書く。

## Risks
- 壊しやすい箇所、不安定な領域、移行中の仕様を書く。
- 一見簡単でも実際には事故りやすいポイントを書く。
- 触る前に確認すべきファイルや手順があれば書く。

## Open Questions
- まだ確定していないことを書く。
- agent が推測で進めず、止まって確認すべき条件を書く。
- 将来見直したい運用ルールがあれば書く。
